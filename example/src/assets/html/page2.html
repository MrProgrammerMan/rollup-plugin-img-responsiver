It also has images, and they too, are responive(this is very cool)

This works because the location of the images as well as this html file have been specified to the img-responsiver plugin.

The plugin only modifies images and html in the specified directories.

This example project has a full config definition in vite.config.ts

You can omit any of the fields, and the default will apply instead.


Below is an image that recides in a folder that isn't specified to the plugin.

It is not responsive, but loads in it's original size and format.

The plugin also works for images with the same name in different directories.
